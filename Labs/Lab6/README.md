### 1. Що таке immutable об’єкт? Чому об’єкти класу String зробили імутабельними? Чому клас String задекларований як final?

**Immutable об’єкт** — це об'єкт, стан якого неможливо змінити після створення. Будь-яка спроба змінити immutable об’єкт створює новий об’єкт із зміненими даними, залишаючи початковий без змін.  

**Чому об’єкти класу String зробили immutable?**  
1. **Безпека:** Оскільки об'єкти String є незмінними, їх можна безпечно передавати між потоками без додаткової синхронізації.  
2. **Оптимізація пам’яті:** Java використовує пул строк (String Pool), що дозволяє зберігати лише одну копію кожного рядка. Immutable властивість гарантує, що рядки в пулі не змінюються.  
3. **Хешування:** Рядки часто використовуються як ключі в хеш-таблицях (наприклад, у `HashMap`). Незмінність дозволяє уникнути проблем із зміною значення ключа після його вставки.  

**Чому клас String задекларований як final?**  
Це запобігає створенню підкласів, які могли б порушити immutable властивість об'єкта, змінюючи його стан.

### 2. Що таке регулярні вирази? Наведіть приклади регулярних виразів.

**Регулярні вирази (Regular Expressions)** — це шаблони для пошуку, перевірки чи заміни тексту у строках. Вони дозволяють працювати з текстовими даними гнучко та ефективно.  

**Приклади регулярних виразів:**  
1. `\d+` — Пошук одного або більше чисел.  
2. `[a-zA-Z]+` — Пошук послідовності літер (верхнього та нижнього регістрів).  
3. `^Hello` — Рядок, який починається зі слова "Hello".  
4. `\s` — Будь-який пробільний символ (пробіл, табуляція, новий рядок).  
5. `.*@.*\..*` — Перевірка наявності електронної адреси.

### 3. У чому різниця між оператором `==` та методом `equals()`?

- **Оператор `==`:**  
  Використовується для порівняння посилань на об’єкти. Повертає `true`, якщо дві змінні вказують на той самий об'єкт у пам’яті.  

- **Метод `equals()`:**  
  Використовується для порівняння вмісту об’єктів. У класі `String`, наприклад, цей метод перевіряє, чи мають два рядки однакові символи в однаковому порядку.

**Приклад:**  
```java
String s1 = new String("hello");
String s2 = new String("hello");

System.out.println(s1 == s2);       // false (різні об’єкти)
System.out.println(s1.equals(s2)); // true  (вміст однаковий)
```

### 4. Для чого потрібні класи StringBuilder та StringBuffer?

Класи `StringBuilder` та `StringBuffer` призначені для роботи зі змінюваними рядками (mutable strings).  
- **StringBuilder:**  
  Використовується для побудови рядків, коли не потрібна синхронізація між потоками. Швидший за `StringBuffer`.  

- **StringBuffer:**  
  Аналогічний `StringBuilder`, але є **потокобезпечним**, оскільки всі його методи синхронізовані. Це робить його менш продуктивним у середовищах, де синхронізація не потрібна.  

**Переваги:**  
- Зменшують накладні витрати на створення нових об'єктів при конкатенації.  
- Ефективні для роботи з великими текстовими даними.

### 5. Як прибрати пробіли на початку та в кінці об’єкту String?

Найпростіший спосіб — використати метод `trim()` класу `String`.  
**Приклад:**  
```java
String str = "   Hello World   ";
String trimmedStr = str.trim(); // "Hello World"
```

Для видалення усіх пробільних символів (включаючи табуляції) можна використовувати метод `strip()` (починаючи з Java 11):  
```java
String strippedStr = str.strip();
```

**Різниця між `trim()` і `strip()`:**  
- `trim()` видаляє лише пробіли (ASCII 32).  
- `strip()` видаляє всі пробільні символи, включаючи нелатинські.
